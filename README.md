# 🏦 CASHIER WATCH v2.4 — Интеллектуальный контроль банковских кассовых операций

**CASHIER WATCH** — это специализированная система видеоаналитики для банковского сектора, предназначенная для автоматизации контроля в зонах обмена валют, оплаты услуг и проведения финансовых транзакций. Система выявляет нарушения регламентов, ошибки при пересчете наличности и подозрительные действия персонала в режиме реального времени.

---

## 📖 Обзор системы
В банковской сфере точность и соблюдение протоколов безопасности имеют критическое значение. **CASHIER WATCH** выступает в роли "цифрового аудитора", который:
*   Следит за корректностью процесса приема/выдачи наличности.
*   Фиксирует факты нарушения регламента обслуживания клиентов.
*   Обеспечивает доказательную базу при разборе спорных ситуаций и претензий.
*   Работает в **закрытом контуре (Air-Gap)**, гарантируя банковскую тайну и безопасность данных.

---

## 🏗 Архитектура и Безопасность
Система спроектирована с учетом жестких требований банковской безопасности и поддерживает два этапа развертывания:

### 1. Этап DEMO (Текущее состояние)
*   **Backend**: Python Notebook (Google Colab) генерирует инциденты.
*   **Transport**: WebSocket сервер с публичным туннелем (ngrok).
*   **Frontend**: GAIS UI подключается к удаленному WebSocket.

### 2. Этап LOCAL (Целевое состояние)
*   **Backend**: Локальный Python сервер (FastAPI/Flask) в закрытом контуре банка.
*   **Transport**: Локальный WebSocket (ws://localhost:8000).
*   **Security**: Поддержка Bearer токенов, SSL/TLS (wss://) и CORS политик.

## 📦 Функциональные модули

### 1. Мониторинг СБ (Security Dashboard)
*   **AI Video Overlay**: Визуализация работы детекторов (руки кассира, пачки купюр, документы).
*   **Live Alerts**: Мгновенные уведомления о критических событиях (например, "Выдача средств до завершения проводки").
*   **Multi-View**: Возможность переключения между камерами разных кассовых узлов.

### 2. Журнал инцидентов (Incident Management)
*   **Карточки инцидентов (Task 4.3)**: Наглядное представление нарушений с цветовой индикацией уровня критичности (Severity-color).
*   **Медиа-контент**: Встроенные превью скриншотов и видеоплееров для быстрого анализа ситуации.
*   **Обнаруженные объекты (Object Tags)**: Список всех объектов, зафиксированных AI-моделью в момент инцидента.
*   **Расширенная фильтрация**: Возможность фильтрации журнала по уровню критичности (Severity), типу нарушения и статусу.
*   **Детальный отчет**: Полноэкранный режим просмотра с подробным описанием и инструментами управления статусом.

### 3. MLOps & ПО (System Stack)
*   **Model Lifecycle**: Управление версиями нейросетей (Детекторы валют, OCR документов, Трекеры).
*   **Service Health**: Мониторинг состояния API-шлюзов и движков инференса.

### 4. Техническая панель (Infrastructure)
*   **Hardware Health**: Контроль температуры GPU и заполнения дисковых массивов (архива).
*   **Network Status**: Мониторинг стабильности RTSP-потоков с IP-камер.

## 🔌 Коннекторы и API (Интеграция)
Система подготовлена к глубокой интеграции в банковскую инфраструктуру через следующие интерфейсы:

### 1. Real-time WebSocket (Task 4.3)
Мгновенная доставка инцидентов из Python-ноутбука или бэкенда в UI.
*   **Протокол**: `ws://` (локально) / `wss://` (через туннель).
*   **Формат**: JSON сообщения типа `NEW_INCIDENT`.
*   **Статус**: Индикатор соединения и счетчик сессии в Header.

### 2. Импорт данных (JSON Import)
Возможность загрузки исторических данных или результатов работы моделей из файлов.
*   **Формат**: Стандартный JSON экспорт из ноутбука.
*   **Интерфейс**: Кнопка "Импорт JSON" во вкладке Инциденты.

### 3. Локальная система (Local OS)
*   **Hardware API**: Сбор метрик нагрузки через системные вызовы (NVIDIA-SMI и др.).
*   **Storage API**: Управление ротацией видеоархива на уровне файловой системы.

## 🚀 Инструкция по запуску

### 1. Фронтенд (React + Vite)
```bash
npm install
npm run dev
```
👉 [http://localhost:3000](http://localhost:3000)

### 2. Бэкенд (WebSocket Server)
Для запуска локального сервера (FastAPI):
```bash
cd backend
pip install fastapi uvicorn pydantic requests
uvicorn server:app --host 0.0.0.0 --port 8000
```

### 3. Подключение Ноутбука (Python)
Используйте скрипт `backend/notebook_client.py` для отправки инцидентов:
```python
from notebook_client import send_incident
send_incident("Касса №4", "Выдача до проверки", "Critical")
```

Подробные инструкции по интеграции и настройке туннелей см. в файле [BACKEND_GUIDE.md](./BACKEND_GUIDE.md).
