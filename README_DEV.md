# Технический отчет и ТЗ для коллег (Cashier Watch v2.4)

## 1. Обзор реализованных изменений

### Фронтенд (React + Vite + Tailwind)
- **Реорганизация MonitoringTab**: Главный экран теперь ориентирован на **инциденты**, а не на видеопоток. 
    - Добавлен дашборд с ключевыми метриками (GPU, FPS, Инциденты).
    - Реализована "Лента событий" (Live Feed) с возможностью выбора инцидента для детального просмотра.
    - Центральное окно отображает скриншот и подробности выбранного нарушения.
- **Терминология**: Полностью удалены ритейл-термины ("товар", "чек"). Внедрена специфика **обменного пункта и кассы услуг** ("купюра", "операция", "валюта").
- **WebSocket**: Настроена интеграция с бэкендом для получения уведомлений в реальном времени.

### Бэкенд (Node.js + Express)
- **Миграция**: Основная логика перенесена с Python (Streamlit) на Node.js для обеспечения масштабируемости и работы WebSocket.
- **Workflow Engine**: Реализован механизм проверки состояний кассира на основе `workflow_config.json`.
- **API**: Созданы эндпоинты для приема событий от ML-скриптов и управления конфигурацией.
- **Логирование**: Внедрена система записи событий в JSONL файлы для последующего анализа.

## 2. ТЗ для ML-разработчиков (Colab / Python)

### Интеграция событий
Для передачи обнаруженного нарушения в UI необходимо отправить POST запрос:

**Endpoint**: `https://<APP_URL>/api/events/handle` (или `/api/incidents/report` для прямого репорта)

**Payload (JSON)**:
```json
{
  "action_id": "CHECK_CASH_DETECTOR",
  "cashier_name": "Касса №4",
  "screenshotUrl": "URL_TO_IMAGE",
  "description": "Кассир пропустил этап проверки на детекторе"
}
```

### Передача метрик
Для отображения нагрузки в UI отправляйте метрики каждые 2-5 секунд:
**Endpoint**: `/api/metrics/system`
```json
{
  "gpu_load": 45,
  "gpu_temp": 68,
  "vram_used": 12.4,
  "vram_total": 24.0,
  "net_in": 85.2
}
```

## 3. ТЗ для DevOps / Системных администраторов

### Развертывание
- **Порт**: Приложение слушает порт `3000`.
- **Node.js**: Требуется версия 18+.
- **Файловая система**: Приложению нужны права на запись в папку `/logs` и `/backend`.

### Конфигурация
- `backend/config.json`: Настройки камер и порогов GPU.
- `backend/workflow_config.json`: Описание графа состояний и разрешенных переходов.
